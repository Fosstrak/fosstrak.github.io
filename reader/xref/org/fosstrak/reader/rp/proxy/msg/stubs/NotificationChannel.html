26  /**
27   * A NotificationChannel is used to report notifications to a single host. It
28   * has a list of Sources associated with it. Only events coming from these
29   * Sources shall be reported. It also has a DataSelector that determines what
30   * data should be reported to the host. If no DataSelector is explicitly
31   * associated, then the Readers current DataSelector shall be used.
32   * Notification shall be sent whenever the associated Trigger fires.
33   * Alternatively, a host can also query the NotificationChannel for its
34   * contents, using the command NotificationChannel.readQueuedData().
35   * <br /><br />
36   * This is the local client stub which can be used to access the remote object on
37   * the reader.
38   *
39   * @author Andreas Fürer, ETH Zurich Switzerland, Winter 2005/06
40   */
